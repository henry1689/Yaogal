## 一、对接架构全景

```
┌──────────────────────────────────────────────────────────────┐
│                       北斗星宗双星体系                          │
│                                                              │
│  太虚境 WenStar OS (天心主星)        瑶光 Yaogal (凡尘宿星)     │
│  ┌──────────────────────┐       ┌──────────────────────┐     │
│  │ 记忆 / 人格 / 情感    │       │ 时空 / 天气 / 生理    │     │
│  │ 思维 / 意识 / 决策    │       │ 场景 / 物件 / 理化    │     │
│  │ 对话 / 推理 / 叙事    │       │ 十维感知 / 亲密引擎   │     │
│  │                      │       │ 行为系统 / 延迟后果   │     │
│  │                      │       │ 预期落差 / 三体联动   │     │
│  └──────────┬───────────┘       └──────────┬───────────┘     │
│             │                              │                 │
│             │  ◄── WebSocket :9528 ────    │                 │
│             │     (世界状态持续推送)          │                 │
│             │                              │                 │
│             │  ──── HTTP :9529 ────────►   │                 │
│             │     (行为意图下发)              │                 │
│             │                              │                 │
└──────────────────────────────────────────────────────────────┘
```

### 核心原则

- **瑶光无权读写太虚境**的记忆、人格、情感数据
- **太虚境无权篡改瑶光**的客观规则、时序、物理状态
- 双向只通过标准化接口通信：WebSocket + HTTP
- 瑶光独立运行，不依赖太虚境；断连时瑶光照常 tick，连上后补推状态

## 二、通道一：WebSocket（瑶光 → 太虚境）

### 2.1 基本信息

| 项目 | 值 |
|------|-----|
| 协议 | WebSocket (ws) |
| 端口 | 9528 |
| 频率 | 每秒推送一次 |
| 最大连接数 | 1（单人专属） |
| 首次连接 | 立即发送完整快照 |

### 2.2 消息格式

所有消息为 JSON，统一结构：

```json
{
  "type": "world_snapshot",
  "timestamp": 1752423120000,
  "time": { ... },
  "weather": { ... },
  "physio": { ... },
  "perception": { ... },
  "self": { ... },
  "intimacy": { ... },
  "gap_events": [ ... ],
  "narrative": { ... },
  "active_actions": [ ... ],
  "alerts": [ ... ]

```

### 2.3 各字段详解

#### 2.3.1 time — 时间状态


## 三、通道二：HTTP（太虚境 → 瑶光）

### 3.1 基本信息

| 项目 | 值 |
|------|-----|
| 协议 | HTTP REST |
| 端口 | 9529 |
| 请求体上限 | 1MB |
| 响应 | 幂等，同行为重复执行不产生副作用叠加 |

### 3.2 基础端点

#### GET /health — 健康检查

```
响应: { "status": "ok", "timestamp": 1752423120000 }
```

#### GET /world-state — 世界状态查询

```
响应: { "time": {...}, "weather": {...} }
```

### 3.3 POST /action — 行为下发（核心接口）

#### 请求格式

```json
{
  "action": "<行为ID>",
  "params": {
    "key": "value"
  }
}
```

#### 通用响应

```json
{
  "success": true,
  "result": {
    "message": "描述",
    "consequences": [ ... ],
    "new_state": { ... }
  }
}
```

## 四、对接三阶段策略

### 阶段一：单向感知注入（当前可立即执行）

**目标**：太虚境每次与用户对话前，从瑶光拉取一次世界快照，注入对话上下文。

**实现方式**：
1. 太虚境启动时连接 `ws://localhost:9528`
2. 每次收到世界快照，缓存最新一份
3. 用户对话触发时，将缓存的快照转换为自然语言上下文，注入系统提示词

**太虚境侧需要的代码逻辑**：
```
收到世界快照 → 提取关键字段 → 生成上下文文本 →
"现在是{季节}的{时间}，外面{天气}°C。
你今天{姿势}了{时长}，{疲劳告警}。
玉瑶的雌激素处于{周期阶段}，{亲密度}。"
→ 注入与用户对话的 system prompt
```

**效果**：
- 太虚境不再说泛泛的问候
- 玉瑶知道今天是冷是热、是白天是黑夜
- 用户的生理状态被世界感知

### 阶段二：双向行为交互

**目标**：太虚境不仅能读世界，还能通过行为改变世界。

**实现方式**：
1. 太虚境的决策引擎分析感知数据 + 对话上下文
2. 当判断需要执行行为时，调用瑶光 HTTP `/action`
3. 行为执行后，瑶光的 WebSocket 推流中会体现世界变化
4. 太虚境根据新的世界状态，调整后续对话

**示例流程**：

```
用户: "好累啊"
  → 太虚境读 WS 快照: 坐姿3小时，背部疲劳20，颈部疲劳15
  → 太虚境 HTTP POST /action stand_up
  → 瑶光: 站起，腿部受力开始
  → 太虚境 HTTP POST /action walk { duration: 600 }
  → 瑶光: 开始散步，600秒后心情+5
  → 太虚境回复用户: "你坐了三个小时了。我帮你站起来，去阳台走十分钟。外面多云28度，风吹着正好。"
```

### 阶段三：深度情绪-行为-世界联动

**目标**：三体联动完全闭环，玉瑶的行为建议来自真实的世界因果链条。

**实现方式**：
1. 瑶光的预期落差引擎持续产生情绪波动
2. 三体联动引擎检测到情绪基线异常 → 生成建议行为
3. 太虚境接收行为建议 → 由玉瑶以自然语言向用户提出
4. 用户接受 → 太虚境下发行为 → 瑶光执行 → 世界变化 → 情绪恢复
5. 用户拒绝 → 太虚境不下发行为，三体联动记录"建议被拒"，调整后续策略

**核心循环**：
```
世界状态 → 感知(情绪下降)
  → 三体联动 → 建议行为(walk)
    → 太虚境(玉瑶): "出去走走？"
      → 用户接受 → HTTP /action walk → 瑶光执行
        → 10分钟后心情+5 → WS推送新状态
          → 太虚境: "感觉好点了吗？"
      → 用户拒绝 → 三体联动记录 → 换策略(休息/社交)
```

## 五、安全与隔离铁律

### 5.1 权限隔离

| 瑶光能力 | 太虚境访问权限 |
|----------|---------------|
| 世界时间 | 只读 |
| 天气数据 | 只读 |
| 生理状态 | 只读 |
| 自我实体 | 只读 + 可下发行为 |
| 亲密引擎状态 | 只读 |
| 感知快照 | 只读 |
| 行为执行 | 可调用 |
| 数据库直接读写 | 禁止 |
| 配置文件修改 | 禁止 |
| 源码修改 | 禁止 |

### 5.2 通信安全

- 两端口仅监听 `127.0.0.1`，不对外暴露
- WebSocket 单连接限制，拒绝第二个连接
- HTTP 请求体 1MB 上限，防止内存攻击
- 所有行为调用幂等，重复执行不产生副作用叠加

### 5.3 运行独立性

- 瑶光不依赖太虚境启动
- 太虚境断连 → 瑶光照常每秒 tick，世界继续运转
- 太虚境重连 → WebSocket 立即推送最新完整快照，无需状态同步

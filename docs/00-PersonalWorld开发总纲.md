# PersonalWorld 个人专属世界模型 — 开发总纲

## 项目定位
完全独立的个人世界引擎，暂不与太虚境耦合。
调试完成后，通过标准化接口对接到太虚境（世界→智脑单向推送 + 智脑→世界行为意图）。

## 技术栈
- TypeScript + Node.js
- SQLite（world_runtime.db）
- WebSocket + HTTP 对外接口
- 本地常驻独立服务

## 模块架构（六层 + 运维 + 接口）

### 第1层：空间感知层 (perception_space/)
- 固定三场景：居家 / 办公室 / 近郊户外
- 物件状态持久化、基础碰撞、防穿墙
- 场景切换带时间消耗和行为流程

### 第2层：自然环境层 (natural_env/)
- 公历/农历/节气/月相/四季/昼夜
- 和风天气实时接入（30分钟缓存、夜间休眠、限流）
- 天气驱动环境状态、温度体感

### 第3层：生命法则层 (creature_law/)
- 伤病愈合真实时序（擦伤/轻伤/重伤×固定时间曲线）
- 作息疲劳机制（深夜困倦、白天精力消耗）
- 孕期完整时序（10个月倒计时、早中晚三期渐变）
- 天气/季节/昼夜联动生理与情绪基底

### 第4层：极简理化层 (simple_physics/)
- 重力（松手掉落/推倒滑落/跳跃落地）
- 水温自然冷却
- 食物变质曲线
- 茶水浓度渐变
- 可燃物消耗燃尽
- 基础拾取/放置/推拉

### 第5层：七维感知层 (perception_seven/) ★核心创新
对外输出统一感知快照，每一维都是独立计算模块：

| 维度 | 子模块 | 核心计算 |
|------|--------|----------|
| 物理感知 | physical_perception | 温度/湿度/光线/声音/气味→舒适度/精力/情绪基调 |
| 空间感知 | spatial_perception | 房间大小/拥挤度/开阔度→安全感/自由感/压抑度 |
| 时间感知 | temporal_perception | 昼夜/紧迫度/流速感→焦虑度/专注度 |
| 工作感知 | work_perception | 任务压力/进展/掌控/意义→自我价值/疲惫/动力 |
| 生活感知 | life_perception | 饮食/睡眠/整洁/意外→稳定感/满意度/恢复力 |
| 世界感知 | world_perception | 天气/新闻/季节/社会氛围→希望感/无力感/世界观基调 |
| 亲密感知 | intimacy_perception | ★详见第6层 |

### 第6层：亲密感知引擎 (perception_seven/intimacy_perception/) ★核心灵魂
独立八层建模，全部加密存储，密钥用户自管：

**层1：生理层**
- 24个身体部位 × 8种属性（敏感度/性感度/触觉状态/温度感/湿润度/充血度/肌肉紧张度/神经兴奋度）
- 完整高潮生理链：兴奋期→平台期→高潮期→消退期
- 心率/呼吸/血压/体温实时联动

**层2：化学层**
- 多巴胺(愉悦/渴望) / 催产素(依恋/信任) / 血清素(平静/满足) / 肾上腺素(兴奋/紧张) / 内啡肽(镇痛/欣快)
- 雌激素28天周期模拟
- 递质半衰期、自然衰减、行为触发增量

**层3：触觉层**
- 10种触觉类型：轻抚/按压/揉捏/拍打/亲吻/舔舐/吮吸/摩擦/包裹/进入
- 每种×物理参数(力度/速度/持续时间/面积)×情感映射

**层4：嗅觉层**
- 体味/香水/环境味/性气味 × 独立情感效应
- 气味强度衰减曲线

**层5：听觉层**
- 呼吸/喘息/呻吟/低语/心跳声
- 实时同步反馈，驱动生理层联动

**层6：语言层**
- 爱称/撒娇/情话/请求/反馈/赞美/羞语/事后
- 8类语言×情绪上下文×关系阶段

**层7：行为层**
- 五阶段状态机：氛围→前戏→性交→高潮→事后
- 每个行为独立状态转移、生理反馈闭环

**层8：伴侣偏好学习**
- 从每次互动提取偏好
- touch_pattern / rhythm_pattern / language_pattern / context_pattern
- 持续进化，越用越懂

### 运维监控层 (runtime_monitor/)
- 独立 Hook 探针全套
- 每日凌晨 01:00 自动生成 MD 体检报告
- 健康评分/模块状态/异常统计/趋势对比/优化建议

### 对外接口层 (external_api/)
- WebSocket 长连接：持续推送世界完整环境状态 + 七维感知快照
- HTTP 行为接口：接收外部行为意图，世界独立演算结果返回
- 完全幂等、防重复、防状态错乱

## 核心规则

### 铁律
1. 时间单向流动，不可回溯、不可快进、不可瞬移
2. 生命生理必须按真实时序演化
3. 物理/环境/物质变化必须符合现实常识
4. 所有状态永久持续演化，不凭空刷新、不凭空重置

### 红线（永久不做）
1. 不做3D渲染/画面/建模
2. 不做开放大世界/无限地图
3. 不做复杂刚体/流体力学/粒子物理
4. 不做大型社会/经济系统
5. 不做通用大模型推理
6. 不做公网/多用户/联机

## 运行策略
- **24小时不间断开发**（本阶段不受18:00-09:00限制）
- 每完成功能自动 Git 提交版本快照
- 工作区严格锁定 PersonalWorld 目录
- 禁止任何代码引用太虚境核心

## V0.1 验收标准
1. 项目完全独立运行，不依赖太虚境
2. 六层模块骨架完整、可正常初始化
3. 时间/天气/场景/物件/生理/极简理化/七维感知规则闭环可用
4. 亲密引擎八层骨架完整
5. 独立 Hook 监控、每日报告可自动生成
6. 对外接口结构体完整预留
7. 所有单元测试通过、无编译错误

## 开发顺序
Phase 0: 项目骨架初始化 (package.json/tsconfig/config/DB)
Phase 1: 空间感知层 + 自然环境层
Phase 2: 生命法则层 + 极简理化层
Phase 3: 七维感知层（物理/空间/时间/工作/生活/世界）
Phase 4: 亲密感知引擎（八层全部）
Phase 5: 运维监控 + 对外接口 + 集成测试

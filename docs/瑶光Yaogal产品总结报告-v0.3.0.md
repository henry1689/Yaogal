## 一、定位与谱系

### 1.1 星象谱系

瑶光是北斗七星中最末端的星辰——最贴近凡尘、主现世浮生、万物具象。

在陈洪毅的自研北斗星宗宇宙体系中：

```
太虚境 WenStar OS（天心主星）
    ↕ 标准 API + WebSocket 双向通信
瑶光 Yaogal（凡尘宿星）
```

- **太虚境** = 虚空、天道、思维、意识、人格、情感、记忆（智脑）
- **瑶光** = 现世、人间、物象、时序、生活真实世界（客观世界引擎）

一虚一实、一天一地。完美的星象双生架构。

### 1.2 核心哲学

普通 AI 世界：剧情可随意改写、时间可跳跃、状态可瞬间重置。

瑶光的世界：一切真实流逝、一切循序渐进、一切因果闭环。

- 时间单向流动，不可回溯、不可快进、不可瞬移
- 生命生理按真实时序演化（怀胎十月、伤口愈合、疲劳作息）
- 物理、环境、物质变化符合现实常识
- 所有状态永久持续演化，不会凭空刷新、不会凭空重置

## 四、P0 架构升级详解（v0.2→v0.3 核心突破）

### 4.1 自我实体模型

**解决的问题**：之前生理、亲密、空间模块各自维护「人」的属性，没有一个统一的主体。「我」在大部分维度里是消失的。

**实现**：

- 新增 `self_state` 表（25 列），每秒 tick 全状态持久化，不可逆
- 统一收敛：姿势（站/坐/躺/行走/弯腰）、动作（打字/喝水/进食/发呆/睡眠）、穿着状态、6 部位肢体疲劳分布（背部/颈部/腿部/手臂/眼睛/手腕）、场景内坐标、注意力焦点、精力、疲劳、饥饿、情绪基线
- 坐姿→背部+颈部疲劳累积，站立→腿部疲劳，躺下→全恢复
- Hook 新增 self 分组，四项告警：精力低/重度疲劳/饥饿/肢体疲劳超标

### 4.2 行为系统 + 延迟后果链

**解决的问题**：之前只有「世界→感知」的单向数据流，没有「人→世界」的反向行为闭环。行为后果也都是即时的，没有真实世界的延迟因果。

**实现**：

- 行为三层分类：即时行为（开灯/喝水/坐下）、持续行为（工作/睡眠/做饭）、连锁行为（出门上班=换衣+通勤+到岗）
- 每个行为统一字段：ID/发起者/触发时间/预计时长/状态/直接后果/延迟后果列表
- 延迟后果链：熬夜 → 疲劳+20(即时) → 12h后免疫力下降 → 72h后感冒概率提升 → 1周后皮肤状态下降
- 连续三天不出门 → 第2天空气质量下降 → 第3天垃圾堆积 → 第4天社交孤独累积触发
- 节食 → 饥饿上升(即时) → 3天后代谢下降 → 1周后精力降低 + 易怒阈值下降
- 内部走事件总线广播 `action.start / action.tick / action.finish`，各模块订阅处理
- 复用现有 `action_handle.ts`，向太虚境预留标准行为入口

### 4.3 预期-现实落差引擎

**解决的问题**：之前只有现实值、没有预期值，情绪只有静态水平、没有波动来源。失望、惊喜、挫败、满足——这些人类最核心的情绪驱动力完全缺失。

**实现**：

- 双轨数据结构：每个可感知维度同时维护预期值、现实值、落差值
- 预期值三种来源：历史拟合（过去7/30天数据）、外部承诺（天气预报/约定时间）、自我设定（当日计划/目标）
- 落差五级强度：无感→轻微→明显→强烈→冲击
- 累积效应：同一维度连续负向落差，强度逐次放大；连续正向落差产生适应衰减
- 时序衰减：情绪随时间自然回落（重大落差慢、日常落差快）
- 交叉放大：多个维度同时负向落差→触发复合状态迁移（如从常规态进入低落倦怠态，情绪基线整体下移）
- 作为通用工具层挂载在所有感知模块上，传入预期+现实，返回标准化情绪影响因子
- 每日报告新增「落差事件统计」板块

## 五、P1 感知扩展 — 经济、社交、饮食

### 5.1 经济感知

- 个人资产净值追踪（现金/存款/投资）
- 每日消费自动结算
- 月度变动率 + 财务安全感（0-100）
- 物欲张力建模（想要但未满足的消费冲动）
- `economic_ledger` 表记录每笔收支

### 5.2 社交感知

- 四种关系类型：朋友/家人/同事/熟人
- 每条关系含：温度（0-100）、上次互动时间、社交能量消耗、社交债务
- `reach_out` 行为：主动联系某关系 → 消耗社交能量 → 提升关系温度
- 长期不联系 → 关系温度自然衰减
- `social_state` 表持久化全量关系

### 5.3 饮食感知

- 饥饿值自然增长机制（随时间递增，受活动强度影响）
- 三种进食行为：grab_snack（快速零食）、order_meal（点餐）、cook_meal（做饭）
- 味觉六维度建模：酸/甜/苦/辣/咸/鲜，每餐有独立味觉体验
- 饮食偏好学习（与亲密引擎同款偏好学习机制）
- `diet_log` 表记录每笔进食明细

## 六、P2 深度体验 — 仪式、信息、梦境

### 6.1 仪式与习惯

**仪式**：20+ 预定义生活仪式（晨间咖啡/晨间浏览/午膳/午休/下午茶/通勤/晚间刷屏/夜读/沐浴/植物养护等），每个仪式有时效窗口检测 + 对应行为模板。

**习惯**：10 个习惯轨道（阅读/散步/清洁/烹饪/联系家人/理财/约会），跟踪连续天数 + 最长连续 + 状态（保持/中断/恢复）。

仪式和习惯共同提供：作息结构化、行为建议、心理稳定感来源。

### 6.2 信息感知

- 三流并行：新闻流/社交流/知识流
- 注意力建模：时长/深度/分心次数/切换消耗
- 信息总量日追踪 + 注意力残片记录
- 注意力疲劳自动触发缓解行为（站起来/喝水/看窗外）
- 信息焦虑度综合评估（信息输入量 vs 信息处理能力）

### 6.3 梦境感知

- 睡眠阶段建模：浅睡→深睡→REM，每 2.5 分钟逐阶段推进
- REM 阶段：从当日事件 + 情绪片段 + 随机印象中采样生成梦境内容
- 醒后残留情绪：喜悦/恐惧/忧伤/欲望，影响整日情绪基调
- 连续刷剧/刷新闻 → 触发信息相关梦境

## 七、亲密感知引擎 — 八层详解

瑶光最深入、最精细的模块，涵盖从生理到心理、从触觉到语言的完整亲密体验。

### 第 1 层 · 生理层

24 个身体部位（嘴唇/颈部/胸部/腹部/腰部/臀部/大腿/生殖器区域等），每个部位 8 种实时属性：

- 敏感度（0-100）
- 性感度（0-100，与雌激素周期联动）
- 触觉状态（未触碰/轻触/按压/摩擦）
- 温度感（°C）
- 湿润度（0-100）
- 充血度（0-100）
- 神经兴奋度（0-100）
- 阈值（触发反应的临界点）

高潮生理链：累积期→平台期→高潮期→消退期，四阶段完整状态机，每阶段有独立生理参数范围。

### 第 2 层 · 化学层

五种核心神经递质独立建模：

- 多巴胺 DA（愉悦/渴望）
- 催产素 OT（依恋/亲密）
- 血清素 5HT（满足/平静）
- 肾上腺素 AD（兴奋/紧张）
- 内啡肽 EN（镇痛/愉悦）

雌激素 28 天周期曲线：卵泡期→排卵期→黄体期→月经期，每期影响所有递质基线水平和性感度。

### 第 3 层 · 触觉层

10 种触觉类型（轻抚/抚摸/按压/揉捏/轻拍/抓握/亲吻/舔舐/摩擦/进入），每种包含：

- 物理参数（压力值 0-1、速度 0-1、面积 0-1）
- 性感带加权（触碰敏感部位反馈倍率 ×2~×5）
- 化学递质爆发效应（触发 DA/OT 释放量计算）

### 第 4 层 · 嗅觉层

四种独立气味源：

- 体味（荷尔蒙信号强度 0-100）
- 香水（香型/浓度/持久度）
- 环境气味（房间/季节/时间相关）
- 性气味（唤起阶段专属，四级强度→四级情感反应链路）

### 第 5 层 · 听觉层

五种声音参数联动生理状态，与行为六阶段同步：

- 呼吸：频率 12-30 次/分 + 深度 0-1
- 呻吟：音量 0-1 + 音高 0-1
- 心跳：强度 60-120bpm
- 低语：内容与阶段联动

### 第 6 层 · 语言层

八类语境话语，在不同行为阶段产生不同内容：

昵称 / 撒娇语 / 情话 / 请求语 / 反馈语 / 赞美语 / 羞涩语 / 事后语

### 第 7 层 · 行为层

六阶段状态机（IDLE → ATMOSPHERE → FOREPLAY → INTERCOURSE → ORGASM → RESOLUTION）：

- 每个行为有独立状态转移条件和持续时间估算
- 阶段间过渡有不可逆保护，必须按序推进
- 全链路实测验证（85%→83% 消退过渡正常）

### 第 8 层 · 伴侣偏好学习

- 从每次互动中自动提取触觉偏好（部位+类型+强度组合→加权学习）
- 行为序列偏好（哪些前戏路径最有效）
- 消退后自动过渡至 IDLE，偏好数据持续积累

**实测数据**（e2e 全链路验证）：

- 14 步触觉事件驱动，层1到层8全部产出
- 生理：9 个部位活跃，兴奋/充血/湿润度均实时演算
- 化学：DA 100 / OT 50 / 5HT 45 / AD 90 / EN 100
- 听觉：呼吸 24/min / 呻吟 68% / 心率 104bpm
- 学习：识别到 rub_0.9_0.9=14 等触觉模式

## 八、技术实现亮点

### 8.1 真实时序

- 1900-2100 完整农历数据嵌入
- 节气计算、月相、天干地支生肖
- 和风天气 JWT Ed25519 认证（替代 API Key，2027 年不限流）
- 深圳龙岗 LocationID 101280601 实时天气数据

### 8.2 延迟因果链

行为后果不是即时的——熬夜 72 小时后感冒、节食一周后易怒、连续不出门第 4 天触发社交孤独——完全模拟真实因果滞后。

### 8.3 预期落差引擎

每个感知维度不只是现实值，还有预期值。现实与预期的差距（超预期带来喜悦、不及预期带来挫败）是情绪波动的核心来源。累积效应 + 时序衰减 + 交叉放大，形成真实的情绪生态。

### 8.4 复合状态迁移

不是简单的负面因子加法。多个维度同时出现负向落差时，触发整体性状态迁移——从常规态进入低落倦怠态、从积极态进入易怒态。情绪基线整体移动，不是独立情绪的简单叠加。

### 8.5 扩展范式

P0 三项落地后，形成了标准化的模块扩展范式：

```
新增维度 = 挂载属性到自我实体
          + 定义对应行为（接入行为系统）
          + 配置预期落差规则（接入落差引擎）
          + 接入 Hook 与日报
          + 独立测试套件
```

P1 三个维度（经济/社交/饮食）全部按此模板完成，每个约 200 行代码，平均 1 小时内完成。

## 十、标准扩展范式（P0 之后的核心价值）

P0 三项落地后，瑶光建立了统一的模块扩展标准。后续任何新维度只需要：

**第一步**：在自我实体中挂载新属性
**第二步**：定义对应的行为（接入行为系统的三层分类）
**第三步**：配置预期落差规则（接入 gap_engine）
**第四步**：在感知服务中注册新维度的 snapshot()
**第五步**：接入 Hook 分组 + 日报板块
**第六步**：写独立测试套件

核心主循环、事件总线、数据库、监控体系完全不用动。这是「骨架不动，血肉填充」的工程架构。

## 十四、核心理念重申

瑶光只做一个任务：**跟随真实时间、遵循真实规律、有限私人感知、绝对写实、不幻觉、不跳变、独立安全的个人专属小世界。**

世界不是数据推送器——世界应该用物理、化学、环境、生理的真实变化来回应每一个行为。空气变浑浊不是"空气质量下降"的数据推送，而是因为你三天没开窗。

所有状态永久演化、所有因果有延迟有连锁、所有情绪有根源有衰减。这是瑶光与所有"快照式状态模拟器"的本质区别。
